


下面是说明书参考 我希望最终的效果是

1. 用户安装的任意全局Python包 都在 /tmp/python_packages 可以直接使用 在任意目录
2. 用户可以使用nodejs的包 安装在/tmp/node_modules
3. 切换到了中国大陆的源 都测试通过了
4. 下面的说明书 目录换成当前这个项目的目录 加上git 我要提交的 让我可以以后自己修改一键导出rootfs

```
 你执行的命令（4 步）

  Step 1：起一个长期运行的"装机容器"

   mkdir -p ~/.tokimo/sandbox

   docker run -dit \
     --name tokimo-builder \
     --platform linux/amd64 \
     debian:12 bash

   用 debian:13（slim 太精简，缺 ca-certificates 等）。要更小可换 ubuntu:24.04。

  Step 2：进去随便装

   docker exec -it tokimo-builder bash

  进去之后你想装什么装什么，参考清单（按需删减）：

   # 基础工具
   apt update && apt install -y \
     curl wget git ca-certificates gnupg lsb-release \
     build-essential pkg-config \
     vim nano less procps htop \
     ripgrep fd-find jq unzip zip \
     python3 python3-pip python3-venv

   # Node 24 + 全局包
   curl -fsSL https://deb.nodesource.com/setup_24.x | bash -
   apt install -y nodejs
   corepack enable
   npm i -g pnpm tsx typescript ts-node nodemon

   # Python 常用包（PEP 668 用 --break-system-packages，或装 pipx）
   pip3 install --break-system-packages \
     python-dotenv requests httpx pydantic ipython rich

   # 清理体积
   apt clean && rm -rf /var/lib/apt/lists/* /tmp/* /var/tmp/*

   exit

  Step 3：导出为 tar（不带 docker 元数据，纯 rootfs）

   docker export tokimo-builder -o ~/.tokimo/sandbox/rootfs.tar
   ls -lh ~/.tokimo/sandbox/rootfs.tar   # 看下大小

  Step 4：解包到目录（bwrap 要的是目录树）

   mkdir -p ~/.tokimo/sandbox/rootfs
   tar -xpf ~/.tokimo/sandbox/rootfs.tar \
     -C ~/.tokimo/sandbox/rootfs \
     --numeric-owner --no-same-owner \
     --exclude='dev/*' --exclude='proc/*' --exclude='sys/*'

   # 验证
   ls ~/.tokimo/sandbox/rootfs       # 应该看到 bin etc usr var ...
   ~/.tokimo/sandbox/rootfs/usr/bin/node --version
   ~/.tokimo/sandbox/rootfs/usr/bin/python3 --version

   不需要 sudo。--no-same-owner 让所有文件归你，bwrap 跑 user namespace 也不冲突。

  收尾：清理 docker 容器（rootfs.tar 也可保留作为备份分发）

   docker rm -f tokimo-builder
   # 团队分发：把 rootfs.tar 上传 OSS / git-lfs / 内网，其他人 tar -xpf 到同一路径即可
